## Why this inventory exists rather than physical folder moves

A physical folder move risks breaking absolute path references in scripts, READMEs, archived snapshots, and historical artifacts. This inventory file provides the same clarity at zero risk:

- HR / recruiters never see Desktop folder structure (they see GitHub + LinkedIn + resume).
- Anyone exploring the Desktop tree sees this README near the top (alphabetical sort puts `_`-prefixed files first) and knows which folders to ignore.
- Future foreman sessions reading this file know not to evaluate drafting folders as portfolio evidence.

If physical folder moves are later desired, they should happen as a separate, deliberate cycle with reference-checking first.
